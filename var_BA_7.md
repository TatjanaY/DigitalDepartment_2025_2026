# 📈 Кейс. Бизнес-аналитика онлайн-ритейла: продажи, ассортимент, удержание, LTV и ROI

## 🎯 Постановка задачи

Компания управляет **интернет-магазином** с широким ассортиментом товаров и международной аудиторией.  
Сервис стремится понять, какие пользователи и категории товаров приносят наибольшую ценность, как развивается поведение клиентов во времени и насколько окупаются маркетинговые вложения.

**Цель исследования** — определить:  
- какие категории товаров и географические сегменты генерируют максимальную выручку;  
- какие пользовательские сегменты демонстрируют высокое удержание и LTV;  
- как меняется активность покупателей со временем;  
- какие маркетинговые каналы наиболее эффективны с точки зрения ROI.

В рамках кейса необходимо провести **комплексный бизнес-анализ** на основе **трёх взаимосвязанных датасетов**, включающий расчёт ключевых продуктовых и маркетинговых метрик: **Retention, Conversion, LTV и ROI**.

**Результатом выполнения задания** должен стать **Jupyter Notebook**, содержащий код, расчёты, функции, визуализации и аналитические выводы.

---

## 📦 Используемые датасеты

В рамках кейса используются **три датасета**, отражающие транзакции, заказы и маркетинговое привлечение.

### 📁 Входные данные (обзор)

---

### 📄 Датасет 1 — Online Retail II  
**Kaggle:** [https://www.kaggle.com/datasets/jillwang87/online-retail-ii](https://www.kaggle.com/datasets/jillwang87/online-retail-ii)  

**Тип данных:** транзакционные данные интернет-магазина  

**Основные признаки:**  
- `Invoice` — номер заказа  
- `StockCode` — код товара  
- `Description` — наименование товара  
- `Quantity` — количество  
- `InvoiceDate` — дата покупки  
- `UnitPrice` — цена за единицу  
- `CustomerID` — уникальный покупатель  
- `Country` — страна  

**Назначение:**  
Используется как **основной источник данных о продажах**, для расчёта **выручки**, **RFM-сегментации**, **частоты заказов**, **LTV** и **когортного анализа**.

---

### 📄 Датасет 2 — Brazilian E-Commerce Public Dataset by Olist  
**Kaggle:** [https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)  

**Тип данных:** данные о заказах бразильского маркетплейса  

**Основные признаки:**  
- `order_id` — идентификатор заказа  
- `customer_unique_id` — уникальный покупатель  
- `order_purchase_timestamp` — дата покупки  
- `order_status` — статус заказа (например, delivered, shipped)  

**Назначение:**  
Дополняет данные по заказам, позволяет **расширить временной охват** анализа, улучшить оценку **retention** и провести **сравнительный анализ поведения** в разных рынках.

---

### 📄 Датасет 3 — Marketing Funnel Dataset  
**Kaggle:** [https://www.kaggle.com/datasets/olistbr/marketing-funnel-olist](https://www.kaggle.com/datasets/olistbr/marketing-funnel-olist)  

**Тип данных:** данные о привлечении пользователей  

**Основные признаки:**  
- `lead_id` — идентификатор лида  
- `first_contact_date` — дата первого контакта  
- `origin` — канал привлечения (organic, paid, referral и др.)  
- `landing_page_id` — точка входа  
- `marketing_cost` — стоимость привлечения  

**Назначение:**  
Используется для анализа **конверсии лида в покупку**, расчёта **CAC** и **ROI по маркетинговым каналам**.

## 🧩 Структура итогового Jupyter Notebook

### 1. 📊 Предварительный анализ данных  
**Цель раздела:** понять структуру, качество и ограничения данных.  

**Необходимо:**  
- вывести размеры таблиц (`shape`);  
- изучить типы данных (`info`);  
- оценить пропуски (особенно в `CustomerID`, `marketing_cost`);

**В конце раздела:** краткие выводы по данным и их пригодности к анализу.

---

### 2. 🧹 Предобработка данных  
**Цель раздела:** подготовить данные к расчёту метрик и объединению.  

**Необходимо:**  
- привести все даты (`InvoiceDate`, `order_purchase_timestamp`, `first_contact_date`) к формату `datetime`;  
- удалить некорректные транзакции (`Quantity ≤ 0`, `UnitPrice ≤ 0`);  
- рассчитать выручку как `Quantity × UnitPrice`;  
- обработать пропущенные значения в `CustomerID` (фильтрация);  
- агрегировать данные по пользователям и временным периодам;  
- объединить датасеты по `CustomerID` / `customer_unique_id` / `lead_id` (через порядковое сопоставление или условное соответствие).  

**В конце раздела:** промежуточные выводы о структуре и чистоте итогового датасета.

---

### 3. 🧠 Разработка функций для анализа данных  
**Цель раздела:** создать переиспользуемый аналитический инструментарий.  

**Ключевое правило:**  
> Все функции — расчёта метрик и визуализации — создаются здесь,  
> но **графики строятся и анализируются в отдельном разделе**.

**В этом разделе разрабатываются функции:**

1. **Для агрегации данных:**  
   - по пользователям;  
   - по товарам / категориям;  
   - по временным периодам (день / неделя / месяц).  

2. **Для фильтрации данных:**  
   - по датам;  
   - по странам (`Country`);  
   - по маркетинговым каналам (`origin`).  

3. **Для расчёта базовых метрик:**  
   - DAU / MAU;  
   - количество заказов;  
   - суммарная и средняя выручка.  

4. **Для формирования пользовательских профилей:**  
   - дата первой/последней покупки;  
   - количество заказов;  
   - суммарная выручка;  
   - lifetime (в днях);  
   - (опционально) RFM-сегментация.  

5. **Для расчёта продуктовых и бизнес-метрик:**  
   - Retention;  
   - Conversion (лид → покупка);  
   - LTV;  
   - CAC и ROI.  

6. **Для визуализации:**  
   - line chart;  
   - bar chart;  
   - heatmap;  
   - funnel chart;  
   - scatter plot.  

**Структура раздела:**  
- краткое введение в модульность и повторное использование кода;  
- реализация функций с аннотациями;  
- перечень функций и их назначение.

---

### 4. 👤 Создание пользовательских профилей  
**Цель раздела:** сегментировать покупателей.  

**Необходимо:**  
- использовать разработанные функции;  
- рассчитать для каждого клиента:  
  - дату первой покупки;  
  - дату последней покупки;  
  - количество заказов;  
  - суммарную выручку;  
  - lifetime;  
- при желании реализовать **RFM-сегментацию** и выделить:  
  - «золотых» клиентов (высокая частота + высокая выручка);  
  - «рискованных» (давно не покупали);  
  - «новичков».  

**В конце раздела:** выводы по выявленным сегментам.

---

### 5. 📐 Расчёт продуктовых и бизнес-метрик  

#### 5.1 🔁 Расчёт удержания (Retention)  
- формирование когорт по дате первой покупки;  
- расчёт retention rate по неделям/месяцам;  
- создание retention-матрицы (когорта × период жизни).  

#### 5.2 🔄 Расчёт конверсии  
- расчёт конверсии:  
  - из лида → в покупку;  
  - по маркетинговым каналам (`origin`);  
  - по странам (`Country`).  

#### 5.3 💰 Расчёт LTV и ROI  
- расчёт LTV по пользователям и когортам;  
- расчёт CAC = `marketing_cost / leads` по каналу;  
- расчёт ROI = (LTV – CAC) / CAC.  

> **Важно:** в этом разделе используются функции, но **визуализация переносится в раздел 6**.

---

### 6. 📉 Визуализация продуктовых и бизнес-метрик  
**Цель раздела:** визуально представить и интерпретировать результаты.  

**Визуализации:**  

1. **🔁 Retention**  
   - Heatmap: X — период жизни, Y — когорта, цвет — retention %;  
   - Line chart: X — время, Y — retention %, линии — когорты.  

2. **🔄 Conversion**  
   - Funnel chart: лид → покупка;  
   - Bar chart: X — каналы, Y — conversion rate;  
   - Stacked bar chart: X — периоды, Y — доля конверсии по странам.  

3. **💰 LTV и ROI**  
   - Line chart: X — lifetime, Y — накопленный LTV по когортам;  
   - Bar chart: X — каналы, Y — ROI;  
   - Scatter plot: X — CAC, Y — LTV, цвет — канал.  

**В конце раздела:** выводы по визуальному анализу — лучшие каналы, устойчивые сегменты, риски оттока.
